Step 0 · 我該從哪裡輸入？

在 Swagger 裡你要自己找 endpoint；在這裡，每個步驟都有輸入框 + 按鈕。你按「送出」後，畫面下方會顯示 API 回傳結果，並解釋每個欄位代表什麼。

你現在要做的事
1. 確認上方連線狀態是 OK
2. 到 Step 1 建立一筆筆記
3. 到 Step 2 輸入問題並看 RAG 回答
這頁 vs Swagger
這頁 = 新手導覽；Swagger = 給已懂 REST 的人直接打 API。
我寫 code 在哪裡？
後端在 GitHub repo 的 src/app/services/langchain_rag.py。先用 UI 理解流程，再對照程式碼。

Step 1 · 建立筆記（REST POST /notes）

RAG 需要「你的資料」。在這一步，你先新增筆記；後端會存進資料庫，並同步到向量資料庫（Chroma）。

預期結果：成功後會看到 201 Created 的 JSON，包含 id。這代表 REST API 建立資源成功。

Step 2 · RAG 問答（LangChain POST /ask）

在這裡輸入你的問題。後端會：① 從筆記中檢索最相關片段 → ② 組成 Prompt → ③ 呼叫 LLM → ④ 回傳答案與來源。

你的問題
→
Retriever 找 top-k 筆記
→
Prompt + LLM
→
answer + sources
目前 LLM 模式：mock。你會看到 [Mock LLM] 開頭的教學式回答；重點請看 sources 是否抓到正確筆記。
answer
LLM 生成的回答。mock 模式是固定教學模板，不是真模型推理。
sources
RAG 檢索到的筆記列表。若為空，代表向量庫還沒資料或問題與筆記無關。
provider
目前 LLM 提供者：mock / ollama / openai
新手檢查清單
✓ sources 是否包含相關筆記？
✓ 新建筆記後，再問同一主題，sources 是否變化？
✓ 切換到 openai/ollama 後，answer 會更像自然語言

Step 4 · Dify 問答（POST /dify/ask）
